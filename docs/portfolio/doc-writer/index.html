文档要素速查
BRD 核心要素
商业背景与机会 → 产品愿景与目标 → 目标用户与市场规模（TAM/SAM/SOM）→ 核心业务需求 → 商业模式与变现路径 → 资源投入与风险评估 → 成功指标（KPI）
PRD 核心要素
背景与目标 → 用户故事（Who/What/Why）→ 功能需求（主流程+边界+异常）→ 非功能需求（性能/安全/兼容性）→ 原型与交互说明 → 验收标准 → 上线计划
OKR 写法原则
O：鼓舞人心、质性描述、不含数字。
KR：可量化、有截止日期、挑战但可达（70%完成率为佳）。
建议每个 O 对应 2-4 个 KR，季度维度为主。
避免：把任务当 KR、KR 互相重叠。
竞品分析框架
① 定位分析（目标用户/核心价值主张）
② 功能对比（核心功能/差异功能/缺失功能）
③ 用户体验评测（Onboarding/核心流程/留存机制）
④ 商业模式 ⑤ 增长策略 ⑥ SWOT ⑦ 机会点
